# Exemplars de design

> **Opcional — apague se o projeto não mantiver `DESIGN.md`.**

Galeria viva de pares **❌ Evite / ✅ Repita**, sempre ancorada num arquivo real
do repo (não exemplo inventado). As **regras** vivem no [`DESIGN.md`](../../DESIGN.md);
aqui mora a **forma**: como a regra se parece em código de verdade, no momento
em que foi violada e corrigida.

## Como usar

- **Ao escrever UI:** procure aqui a regra antes de aplicar. Se já tem exemplar,
  copie o lado ✅.
- **Ao revisar:** achou violação nova de uma regra do `DESIGN.md`? Conserte e
  **adicione entrada aqui** com antes/depois real. Achado de design não morre
  na conversa — vira exemplar (ou regra de lint, ou eval).
- **Formato de cada entrada:** título, regra citada do `DESIGN.md`, snippet ❌
  com arquivo de origem, snippet ✅ corrigido, porquê em uma linha.
